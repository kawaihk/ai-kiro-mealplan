## 4. 対応優先度別 次のステップ

### 高優先度（設定変更で大幅改善）

1. **C003**: `lombok.config` に `lombok.addLombokGeneratedAnnotation = true` を追加し、JaCoCo の `pom.xml` 設定に `@lombok.Generated` 除外を追加する。これにより model・entity パッケージのカバレッジが大幅に改善される。

2. **C001**: `AiMealPlanApplication` を JaCoCo 除外設定に追加する。

3. **C002**: `SecurityConfig` を JaCoCo 除外設定に追加する。

### 中優先度（テスト追加）

4. **C004**: `UserServiceImplTest` に `updateUser_roleIsNull` テストを追加する。
5. **C005**: `MealServiceImplTest` に `getMealsByMealPlanId_emptyList` テストを追加する。
6. **C006**: `PfcRatioSumValidatorTest` に部分 `null` ケースのテストを追加する。
7. **C012**: `UserDtoTest` に setter・`equals` 分岐テストを追加する。

### 低優先度（C003適用後に残存する場合）

8. **C007〜C011**: Lombok除外設定後も残るカバレッジ不足があれば個別対応する。

## 5. Lombok除外設定の具体的な手順（C003対応）

### Step 1: `lombok.config` をプロジェクトルートに作成

```properties
# lombok.config
lombok.addLombokGeneratedAnnotation = true
```

### Step 2: `pom.xml` のJaCoCo設定に除外を追加

```xml
<configuration>
    <excludes>
        <exclude>**/AiMealPlanApplication.class</exclude>
        <exclude>**/config/SecurityConfig.class</exclude>
    </excludes>
</configuration>
```

`@lombok.Generated` が付与されたメソッドはJaCoCoが自動的に除外するため、Step 1の設定だけでLombok生成コードの除外が完了する。
